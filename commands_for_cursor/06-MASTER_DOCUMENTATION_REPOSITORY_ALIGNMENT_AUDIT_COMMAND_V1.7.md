# 14. SUCCESS CRITERIA

The task is complete only if:

- no production code is modified;
- no tests are modified;
- no existing documentation is edited;
- all required audit outputs are created;
- all documentation claims are audited;
- command version alignment is audited;
- superseded commands are identified;
- README, Docs index and feature matrix status are assessed;
- unsupported claims are listed;
- documentation remediation plan is produced;
- physical/external QA claims are verified or marked unsupported;
- final git status confirms only audit outputs changed.

Do not commit or push automatically.

Stop after producing the documentation/repository alignment audit outputs and final summary.
